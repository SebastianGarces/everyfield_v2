### A2 — Rate limiting on login & register ✅ (AC2.1–2.5)
- `src/lib/auth/rate-limit.ts` (new): Postgres-backed via `auth_attempts` (no in-memory state). Login: ≥5 failed/email/15min or ≥20 failed/IP/15min. Register: ≥3 failed/IP/hour. Only failures count toward limits (success resets nothing because it never accumulates). IP from `x-forwarded-for` first hop; IP checks skipped when unavailable. 1-day opportunistic cleanup piggybacked on writes.
- Wired into `src/app/(auth)/login/actions.ts` (check **before** user lookup/password verify; generic "Too many attempts" message — no account-existence leak) and `src/app/(auth)/register/actions.ts`.
- **Deviation from plan:** the spec named `src/lib/auth/actions.ts` as the login action, but that file only contains `logout()`. The real login action is `src/app/(auth)/login/actions.ts` — that's what was edited.
